# BrowserStudio v1.7.25

## 钱包批量导入修复

- 导入 Rabby、Jupiter 或 MetaMask 前不再要求用户手动启动目标环境。
- CSV 模板现在写入环境编号、Profile ID、环境名称和数据文件夹 `storage_id`；用户只需填写 `mnemonic` 列。
- 选择文件和执行导入时都会交叉核验环境映射，避免编号变更、文件夹变更或模板误改后导入到错误环境。
- 客户端会自动启动停止的目标环境，导入完成后保持环境窗口打开。
- 修复前端同时“要求已启动”又“拒绝已启动”导致开始导入无法正常使用的逻辑矛盾。

## 安全与兼容性

- 助记词仍只保留在一次性后端内存会话中，不返回前端、不写入数据库或日志，15 分钟过期后清除。
- 本地解锁密码仅用于当次导入，不落盘。
- 保留旧版 CSV/TXT 的精确 Profile ID、环境编号或名称映射兼容能力；新下载的模板使用更严格的四项核验。

## data 数据恢复

- 旧 `data` 恢复改为以浏览器数据文件夹名作为主要映射身份，不再把环境编号作为恢复冲突条件。
- 备份数据文件夹与当前已创建环境同名时，可以明确预览并覆盖该环境的浏览器数据。
- 覆盖只替换账号、Cookies、Local Storage 和扩展本地数据，保留当前环境的 Profile ID、名称、内核、代理和用户自定义编号。
- 覆盖前会将当前数据目录移入 `data/recovery-backups/.../overwritten-data` 回滚备份；复制或数据库写入失败时会自动恢复原目录。
- 恢复后可先检查账号和 Cookies，再安装官方钱包扩展并执行助记词批量导入。
