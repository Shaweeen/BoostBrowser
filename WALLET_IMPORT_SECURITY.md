# Wallet batch-import security boundary

BrowserStudio supports one-way mnemonic import into fresh official Rabby,
Jupiter, and MetaMask extension profiles. It does not expose an API or user
interface for exporting a mnemonic or private key.

## Enforced controls

- Only these exact extension IDs are accepted:
  - Rabby: `acmacodkjbdgmoleebolmdjonilkdbch`
  - Jupiter: `iledlaeogohbilgbfhmbgkgmpplbfboh`
  - MetaMask: `nkbihfbeogaeaoehlefnkodbefgpgknn`
- The CSV/TXT file is opened by the Go backend. Preview data sent to the UI
  contains only row number, environment ID/name, word count, and running state.
- Secret rows are held in a one-use in-memory session for at most 15 minutes.
  Execute, cancel, expiry, and application shutdown clear the session fields.
- Mnemonics and import passwords are not written to BrowserStudio databases,
  configuration, local storage, result objects, or application logs.
- Text is entered with Chrome DevTools `Input.insertText`; the operating-system
  clipboard is never used.
- Import refuses an already-initialized wallet instead of overwriting it.
- Environments in the batch are locked against concurrent manual startup.
- The import browser suppresses ordinary startup pages and uses
  `--disable-extensions-except` so only the selected wallet extension can run
  during secret entry.
- Results contain status and, where the wallet UI exposes it, a public address.

## Residual risks

- A wallet extension necessarily receives the mnemonic and stores equivalent
  encrypted key material in its own profile. BrowserStudio cannot make a wallet
  extension import a secret while also preventing that extension from receiving
  the secret.
- Jupiter Wallet is currently closed source. Its published package and official
  identity can be checked, but its complete internal behaviour cannot be proven
  independently. Users must trust Jupiter's official extension publisher.
- MetaMask and Rabby are open source, but the installed package and all future
  updates remain part of the trusted computing base.
- Malware, an administrator/debugger on the same PC, a compromised browser
  binary, memory inspection, screen capture, and keystroke capture are outside
  the protection boundary.
- Go and Chromium may retain transient copies in process memory until reclaimed;
  field clearing reduces lifetime but is not a cryptographic secure-erase
  guarantee.
- The user-supplied CSV/TXT file remains on disk until the user securely removes
  it. Templates generated by BrowserStudio contain no secrets.

For high-value wallets, use a clean offline PC, verify official extension IDs,
disconnect unnecessary software, import one wallet first, and securely delete
the completed mapping file after verification.
