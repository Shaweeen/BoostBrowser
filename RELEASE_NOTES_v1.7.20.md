# BrowserStudio v1.7.20

## 主要更新

- 修复全新 Windows 用户电脑点击钱包批量导入的“选择 CSV/TXT”后无法打开文件选择窗口的问题。
- 原生 Wails 文件选择器不可用时，自动切换到 Windows 内置 WinForms 文件选择器，不要求用户安装 Git、GitHub CLI 或开发工具。
- 后备选择器只返回文件路径；CSV/TXT 中的助记词仍只由 Go 后端读取并保存在一次性内存会话中，不进入前端、数据库或日志。
- 修复 Wails 后端以字符串返回错误时，界面只显示“读取导入文件失败”而隐藏真实原因的问题。

## 升级说明

- 已安装 v1.7.19 或其他 v1.7.x 的用户可直接在客户端在线升级到 v1.7.20，不需要卸载或重新安装。
- 在线升级不会覆盖环境、浏览器数据、代理池、扩展、内核、配置或激活状态。
- 如果旧客户端更新器异常，可使用 `BrowserStudio-Repair-Upgrade-v1.7.20.ps1` 执行无损修复升级。

## 兼容性

- Windows 10/11 x64。
- Windows 后备文件选择器使用系统自带的 Windows PowerShell 与 .NET WinForms，不需要额外安装软件。
