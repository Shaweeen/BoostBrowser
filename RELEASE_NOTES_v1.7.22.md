# BrowserStudio v1.7.22

## 主要更新

- 修复同步助手中主控浏览器二级、三级 Chrome 原生菜单超出平铺窗口后，鼠标移动与点击映射不准确的问题。
- 同步启动后持续识别主控及所有跟随环境的 Chrome 菜单、子菜单、确认框和扩展弹窗，并约束在各自排列窗口范围内。
- 弹层超过当前平铺单元尺寸时自动缩小到可用范围；只越过窗口边缘时自动回移，不改变已经位于窗口内的正常弹层。
- 子菜单通过完整 Win32 owner 链归属到对应环境，避免二级、三级弹层误配到其他浏览器。
- 弹层调整使用 `SWP_NOACTIVATE`，不会抢夺主控焦点；普通浏览器主窗口、DevTools 和输入法辅助窗口不会被误缩放。
- 修复同步 Hook 启动失败时后台输入队列未及时停止的问题。

## 升级说明

- 本版本包含 v1.7.21 的钱包 CSV 环境编号识别修复。
- 已安装任意 v1.7.x 的用户可以直接在线升级到 v1.7.22，不需要依次安装 v1.7.20 或 v1.7.21。
- 升级不会覆盖环境、浏览器数据、代理池、扩展、内核、配置或激活状态。
- 如果旧客户端更新器异常，可使用 `BrowserStudio-Repair-Upgrade-v1.7.22.ps1` 执行无损修复升级。

## 兼容性

- Windows 10/11 x64。
- 弹层约束仅在同步功能运行期间生效，停止同步后不再调整浏览器菜单位置。
