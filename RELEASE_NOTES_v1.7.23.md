# BrowserStudio v1.7.23

## 本次更新

- 新增“从旧版 data 恢复”：可直接识别旧安装或备份中的 `data/app.db`，预览环境编号与数据目录状态，再将环境、浏览器账号、Cookies、Local Storage 和扩展本地数据非覆盖导入当前版本。
- 恢复过程不会替换当前客户端程序和功能；相同环境 ID、相同编号或已有目标目录会自动跳过，并在操作前创建当前数据库回滚备份。
- 支持 100GB 以上旧数据按环境逐项复制和进度显示；单项失败会清理未完成目标，不覆盖旧数据或当前数据。
- 钱包批量导入改为匹配 CSV 环境编号后，只向已启动且调试接口就绪的环境导入；导入完成后环境保持打开。
- 钱包密码由用户在每次导入时自定义，文件、助记词与密码不写入数据库或日志；仍拒绝覆盖已经初始化的钱包。
- 环境批量创建与钱包导入单批上限提升到 2000，覆盖 500+ 环境使用场景。
- 修复钱包并发导入进度的数据竞争，并在导入期间阻止目标环境被客户端关闭。

## 恢复使用方法

1. 打开“系统设置 → 配置备份与恢复 → 从旧版 data 恢复”。
2. 选择旧版 `data` 文件夹，或包含该文件夹的旧安装目录。
3. 检查编号、冲突和目录缺失预览；关闭全部浏览器环境后执行恢复。

精确恢复必须存在旧版 `data/app.db`。只有浏览器文件夹而没有数据库时无法可靠还原环境编号，因此客户端不会猜测导入。

## 钱包批量导入

1. 创建所需环境并安装官方 Rabby、Jupiter 或 MetaMask 扩展。
2. 启动 CSV 中指定编号的全部环境，并保持窗口打开。
3. 在扩展管理中选择钱包批量导入文件，设置本批次自定义本地解锁密码并执行。

导入已启动环境时，请关闭无关网页并停用不可信扩展。钱包扩展会按自身机制保存加密密钥；Jupiter 为闭源第三方扩展，BrowserStudio 无法审计其内部行为。

## 升级兼容性

- Windows 10/11 x64。
- 已安装任意 v1.7.x 的用户可以直接在线升级到 v1.7.23，无需逐级安装。
- 在线升级和修复升级都会保留当前 `data`；新版旧数据恢复入口用于卸载重装、路径迁移或手工备份恢复。
